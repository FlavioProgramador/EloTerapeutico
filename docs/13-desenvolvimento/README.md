## Princípios

- entender o contrato existente antes de alterar;
- manter isolamento e confidencialidade;
- não misturar refatoração ampla com correção funcional;
- migrations, testes e documentação acompanham a mudança;
- commits pequenos em português;
- nenhuma alteração direta na `main`;
- nenhum segredo ou dado real.
