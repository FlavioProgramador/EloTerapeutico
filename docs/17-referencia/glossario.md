# Glossário

- **Aditivo:** complemento imutável associado a uma evolução clínica já bloqueada ou que precisa de retificação.
- **Anamnese:** conjunto estruturado de informações clínicas iniciais e seu histórico de versões.
- **Asaas:** gateway usado para clientes, cobranças, assinaturas e webhooks.
- **Audit log:** registro de ação, usuário, horário, objeto técnico, IP e user agent.
- **Billing:** cobrança da assinatura do próprio produto; não confundir com o financeiro do terapeuta.
- **ClinicalExport:** job persistido no banco para gerar exportação clínica.
- **Evolução:** registro clínico de uma sessão, com conteúdo criptografado e possível confidencialidade.
- **Financeiro:** receitas, despesas, mensalidades e pagamentos vinculados à operação do profissional.
- **JWT:** token assinado usado pela API para autenticação.
- **Prontuário:** conjunto de anamnese, evoluções, metas, documentos, formulários e exportações de um paciente.
- **Role:** papel funcional do usuário: terapeuta, secretária ou administrador.
- **Tenant:** organização lógica que isola usuários e dados. Não existe como entidade explícita no código analisado.
- **Worker:** processo separado que busca e processa jobs de exportação no banco.
