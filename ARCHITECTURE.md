# Architecture Overview – Elo Terapêutico

This document provides a high‑level view of the system’s components, their responsibilities, and how they interact in both **development** and **production (Azure)** environments.

---

## 1️⃣ Components
| Component | Language / Framework | Responsibility |
|-----------|----------------------|----------------|
| **Backend API** | Django 5.x + DRF | Business logic, authentication (SimpleJWT), data model, encryption, audit‑trail, API endpoints. |
| **Frontend** | Next.js 14 (App Router) + TypeScript + Tailwind CSS | UI/UX, client‑side state, calls to the backend, SSR for SEO‑friendly pages. |
| **Database** | PostgreSQL (Azure Flexible Server or VM) | Persistent storage for users, patients, appointments, medical records, financial transactions. |
| **Blob Storage** | Azure Blob Storage | Secure storage for uploaded documents, PDFs, images attached to patient records. |
| **CI/CD** | GitHub Actions | Runs tests, lint, builds Docker images, and deploys to Azure App Service / Azure Static Web Apps. |
| **Auth / Secrets** | Azure App Service Configuration | Stores `SECRET_KEY`, `DATABASE_URL`, `JWT_SECRET`, `AZU…_CONNECTION_STRING` securely (no source‑code exposure). |

---

## 2️⃣ Interaction Flow (Development)
1. **Developer** runs `npm run dev` (frontend) and `python manage.py runserver` (backend). Both services talk via **http://localhost:8000**.
2. Frontend uses **Axios / fetch** with JWT stored in HttpOnly cookies to call protected endpoints.
3. Sensitive fields (`Evolution.content`) are encrypted in the Django layer before persisting to PostgreSQL.
4. Files uploaded via the frontend are sent to a **signed SAS URL** generated by a backend endpoint and stored in Azure Blob Storage.

---

## 3️⃣ Interaction Flow (Production on Azure)
```
Client (browser) → Azure Front Door / CDN (optional) → Next.js App Service (Node)
                                            ↕
                                   Calls Backend API Service (Django App Service)
                                            ↕
                                 PostgreSQL (Azure Flexible Server or VM)
                                            ↕
                                 Azure Blob Storage (documents)
```
* **TLS** is enforced end‑to‑end.
* **App Service** env variables hold all secrets.
* **Health probes** ensure the API restarts automatically on failures.

---

## 4️⃣ Security Highlights
* **HTTPS** everywhere (HSTS enabled).
* **JWT** stored in `HttpOnly` + `Secure` cookies.
* **Field‑level encryption** via `django‑cryptography`.
* **Network security groups** restrict DB access to the App Service only.
* **Audit logs** written to a dedicated `audit_log` table (immutable).

---

## 5️⃣ Extensibility
* New micro‑services (e.g., analytics, notifications) can be added as separate Docker containers and hooked through Azure Service Bus or HTTP APIs without breaking existing contracts.
* The OpenAPI spec (`openapi.yaml`) lives under `backend/docs/` and can be versioned for external partners.
