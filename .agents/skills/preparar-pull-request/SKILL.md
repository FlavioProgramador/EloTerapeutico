# Preparar Pull Request
1. Confirme o escopo e revise `git status` e diff.
2. Identifique arquivos fora do escopo.
3. Execute formatter, lint, typecheck, testes e build relevantes.
4. Verifique migrations e documentação.
5. Não faça push, merge ou deploy sem autorização.
6. Sugira branch, Conventional Commit, título do PR, descrição, checklist, riscos e rollback.
7. Declare qualquer teste não executado.
