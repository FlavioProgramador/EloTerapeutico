# Documentar entrega
1. Leia `.agents/agents.md` e atue como `@devops`.
2. Compare documentação com código real.
3. Atualize instalação, variáveis, execução, Docker, migrations, testes, lint, build, endpoints e troubleshooting quando aplicável.
4. Mantenha `.env.example` sem segredos.
5. Teste os comandos quando seguro.
6. Não faça deploy nem documente planejamento como concluído.
7. Informe comandos validados e limitações.
