# Planejar módulo
1. Leia `.agents/agents.md` e atue como `@architect`.
2. Inspecione stack, código, documentação e testes existentes.
3. Identifique o que existe, lacunas, dependências e arquivos compartilhados.
4. Crie `docs/modules/<modulo>-spec.md` com objetivo, escopo, permissões, requisitos, regras, fluxos, modelos, endpoints, validações, estados da UI, acessibilidade, riscos, testes, migrations, arquivos previstos, divisão em PRs e critérios de aceite.
5. Não altere código de produção.
6. Apresente o resumo e pare para aprovação humana.
